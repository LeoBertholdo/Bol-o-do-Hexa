## 🔴 Importante — resolver antes da Copa

### 1. O nome da função do robô tem um traço a mais (`api-football-sync-`)

**Onde:** `bolao2026.html` linha 4361, `ROBO_SETUP.md` linhas 53/136/156, `API_FOOTBALL_TESTE.md` linha 20.

O código, a documentação e o agendador (cron) chamam a função com o nome **`api-football-sync-`** (com traço no fim). Mas a pasta do código-fonte se chama **`api-football-sync`** (sem traço):

```
supabase/functions/api-football-sync/   ← pasta real (sem traço)
```

Quando se publica uma função pelo método padrão do Supabase, o nome dela é o nome da pasta. Então, se a função foi publicada como `api-football-sync`, **toda chamada a `api-football-sync-` vai falhar** (função não encontrada) — o robô de placares ao vivo nunca roda, e o botão "atualizar agora" do app sempre dá erro.

O traço no fim é quase certamente um engano de digitação que se espalhou para todos os lugares.

**O que fazer:** decidir UM nome e usar o mesmo em todo lugar. O mais simples é tirar o traço — usar `api-football-sync` no código (linha 4361), no `ROBO_SETUP.md` e no cron. Se a função já estiver publicada no Supabase **com** o traço, então o certo é renomear a pasta para bater. O importante é que os três (pasta publicada, app, cron) usem exatamente o mesmo nome.

### 2. A aba "Brasileirão" no menu é, na verdade, um laboratório de testes

**Onde:** `bolao2026.html` linha 2573 (botão do menu) → leva ao bloco marcado "BRASILEIRAO TEST LAB" (CSS linhas 1477–2045, JavaScript linhas 4358–5872).

O próprio código avisa, em comentário: *"remove this block after the Copa integration is ready"* (remova este bloco quando a integração da Copa estiver pronta). É um ambiente para testar placares automáticos usando o Brasileirão, mas hoje ele aparece como uma aba normal chamada **"Brasileirão"** para qualquer pessoa que entre no site.

Consequência: ~1.700 linhas de JavaScript + ~570 de CSS de código temporário estão no app em produção, e os participantes veem uma aba de teste no meio do bolão.

**O que fazer:** se ainda estiver testando o robô, tudo bem manter — mas pelo menos renomeie/esconda a aba para não confundir os participantes. Quando o teste terminar, remova os dois blocos marcados (`TEST LAB ... END TEST LAB`) e o botão da linha 2573 de uma vez só.

### 3. Os placares da Copa NÃO são preenchidos automaticamente

**Onde:** todas as referências a `live_scores` no app (linhas 4794, 5051, 5070, 5846) estão **dentro** do bloco de teste do Brasileirão.

O robô de placares ao vivo e a leitura da tabela `live_scores` só estão ligados ao laboratório do Brasileirão. **O bolão da Copa em si não consome placares automáticos** — os resultados dos jogos da Copa precisam ser digitados manualmente pelo admin. Isso é coerente com o comentário "a integração da Copa ainda não está pronta", mas é importante você saber: no dia dos jogos, alguém vai ter que lançar os placares à mão.

**O que fazer:** nenhuma correção obrigatória — só ter clareza de que digitar resultado é trabalho manual, a menos que você termine a integração (conectar o `live_scores` ao bolão principal como já está feito no laboratório).

## 🟠 Médio — qualidade e dados

### 4. Tabelas de dados deixadas vazias (placeholders nunca preenchidos)

**Onde:** `bolao2026.html` linhas 2653–2655.

```js
const TEAM_CONDUCT_SCORES = Object.freeze({});   // vazio
const FIFA_RANKING_ORDER  = Object.freeze({});   // vazio
const FIFA_RANKING_HISTORY = Object.freeze([FIFA_RANKING_ORDER]); // = [ {} ]
```

Esses três objetos deveriam conter, respectivamente, a pontuação de fair-play de cada seleção e o ranking da FIFA. Estão vazios. Eles são usados como critérios de desempate na classificação dos grupos (`compareTeams`, linha 4163, e `rankEqualPointRows`, linha 4146).

Como estão vazios, esses dois critérios **não fazem nada** — sempre dão empate. Quando duas seleções terminam empatadas em pontos, saldo, gols e confronto direto, o desempate cai direto para a **ordem alfabética** do nome do país, em vez do ranking FIFA (que é a regra oficial de desempate da FIFA). Isso só importa num cenário de empate muito específico, mas pode definir quem avança de forma diferente da realidade.

**O que fazer:** ou preencher as tabelas com os números reais (ranking FIFA e fair-play das 48 seleções), ou, se você não quer usar esses critérios, remover as funções relacionadas para deixar claro que o desempate final é alfabético.

### 5. Confira os confrontos, grupos e horários da Copa contra a fonte oficial

**Onde:** `bolao2026.html` linhas 2589–2704 (`GROUPS`, `OFFICIAL_GROUP_FIXTURES`, `OFFICIAL_KO_FIXTURES`).

Os 12 grupos, as 48 seleções, os 72 jogos da fase de grupos e o chaveamento do mata-mata estão todos fixados no código, com data e hora (em UTC) de cada jogo. A estrutura está internamente consistente e a tabela de alocação dos 8 melhores terceiros é o formato oficial real da FIFA — ou seja, **não parece inventado**. Mas como esses dados foram montados por IA, é exatamente o tipo de coisa que pode ter um confronto trocado, uma seleção errada ou um horário furado.

**O que fazer:** antes da Copa, conferir item por item contra o site oficial da FIFA — principalmente os **horários de início**, porque é o horário que tranca os palpites de cada jogo. Um horário errado pode fechar (ou deixar aberto) um palpite na hora errada.

**O que fazer:** nada urgente — código morto não causa bug, só deixa o arquivo maior e mais difícil de manter. Vale remover numa limpeza, com calma, testando que nada quebra.

## 🟡 Pequeno — polimento

### 8. Dois "engole-erros" silenciosos

**Onde:** `bolao2026.html` linhas 2806 e 2842 — `}catch(e){}`.

Quando dá erro nesses dois pontos (provavelmente leitura de backup / localStorage), o app simplesmente ignora sem registrar nada. Se um dia um participante disser "meus dados sumiram", não vai haver pista nenhuma para investigar.

**O que fazer:** trocar por `}catch(e){console.warn(e);}` — não muda o comportamento para o usuário, mas deixa rastro no console para depuração.
